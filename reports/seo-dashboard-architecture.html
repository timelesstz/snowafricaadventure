SEO Dashboard Architecture

A production-grade SEO monitoring system built with Next.js App Router, Prisma, and Google APIs. This document covers every layer — database models, service libraries, API routes, admin UI, and cron jobs — so you can port it to any Next.js project.

Next.js 16 Prisma 7 Google APIs TypeScript Cheerio Tailwind CSS 4

1 System Overview

The SEO Dashboard is a 5-layer system that syncs data from Google Search Console and Google Analytics 4 into a local database, then surfaces insights through an admin UI with health scoring, on-page audits, keyword tracking, 404 monitoring, and actionable recommendations.

💾
Database Layer

9 Prisma models + 4 enums storing GSC queries, GA4 metrics, page audits, keywords, 404s, redirects, and sync logs.

⚙
Service Layer

8 library modules: Google auth, GSC client, GA4 client, page auditor, recommendation engine, query advisor, bot detector, redirect cache.

🌐
API Layer

15+ API routes for dashboard data, manual syncs, page audits, keyword management, 404 tracking, and redirect CRUD.

📊
UI Layer

8 admin pages with health gauge, metric cards, sparklines, search query tables, and recommendation cards.

⏰
Cron Layer

Daily 4 AM UTC sync job pulling last 3 days of GSC data and yesterday's GA4 organic traffic.

🛡
SEO Utilities

14 JSON-LD schema generators, dynamic sitemap, smart robots.txt, and metadata helpers.

6 Google Auth

Service account authentication for both GSC and GA4 APIs. Reads a base64-encoded JSON key from environment variables and caches the auth instance.

src/lib/seo-dashboard/google-auth.ts TypeScript 
import { google, Auth } from "googleapis";

let cachedAuth: Auth.GoogleAuth | null = null;

function getServiceAccountKey(): Record<string, string> | null {
  const encoded = process.env.GOOGLE_SERVICE_ACCOUNT_KEY;
  if (!encoded) return null;
  try {
    return JSON.parse(Buffer.from(encoded, "base64").toString("utf-8"));
  } catch {
    console.error("Failed to parse GOOGLE_SERVICE_ACCOUNT_KEY");
    return null;
  }
}

export function getGoogleAuth() {
  if (cachedAuth) return cachedAuth;
  const key = getServiceAccountKey();
  if (!key) throw new Error("GOOGLE_SERVICE_ACCOUNT_KEY not set");

  cachedAuth = new google.auth.GoogleAuth({
    credentials: key,
    scopes: [
      "https://www.googleapis.com/auth/webmasters.readonly",
      "https://www.googleapis.com/auth/analytics.readonly",
    ],
  });
  return cachedAuth;
}

export function getSearchConsoleClient() {
  return google.searchconsole({ version: "v1", auth: getGoogleAuth() });
}

export function getAnalyticsDataClient() {
  return google.analyticsdata({ version: "v1beta", auth: getGoogleAuth() });
}

export function isGoogleConfigured(): boolean {
  return !!getServiceAccountKey();
}

export function getServiceAccountEmail(): string | null {
  return getServiceAccountKey()?.client_email ?? null;
}
Google Setup Requirements 1. Create a Google Cloud project with Search Console API and Analytics Data API enabled.
2. Create a service account and download the JSON key.
3. Base64-encode the JSON key: base64 -w0 service-account.json
4. Set as GOOGLE_SERVICE_ACCOUNT_KEY env var.
5. Add the service account email as a user in GSC and GA4.

7 GSC Client

Fetches search analytics data from Google Search Console. Handles pagination (25K row limit per request) and returns structured rows.

src/lib/seo-dashboard/gsc-client.ts TypeScript 
import { getSearchConsoleClient } from "./google-auth";

// Key function: fetches query-level data with pagination
export async function fetchSearchAnalytics(
  siteUrl: string,
  dateRange: DateRange,
  rowLimit = 25000
): Promise<GscRow[]> {
  const client = getSearchConsoleClient();
  let startRow = 0;

  while (true) {
    const response = await client.searchanalytics.query({
      siteUrl,
      requestBody: {
        startDate: dateRange.startDate,
        endDate: dateRange.endDate,
        dimensions: ["query", "page", "date", "country", "device"],
        rowLimit: Math.min(rowLimit, 25000),
        startRow,
      },
    });
    // ... parse rows, break when exhausted
  }
}

// Also exports: fetchPageMetrics(), listSites()

8 GA4 Client

Fetches organic search traffic filtered to the "Organic Search" channel. Returns sessions, users, bounce rate, and conversions by landing page.

src/lib/seo-dashboard/ga4-client.ts TypeScript 
export async function fetchOrganicTraffic(
  propertyId: string, dateRange: DateRange
): Promise<OrganicRow[]> {
  const client = getAnalyticsDataClient();

  const response = await client.properties.runReport({
    property: `properties/${propertyId}`,
    requestBody: {
      dateRanges: [{ startDate, endDate }],
      dimensions: [
        { name: "date" },
        { name: "landingPagePlusQueryString" },
      ],
      metrics: [
        { name: "sessions" },
        { name: "totalUsers" },
        { name: "newUsers" },
        { name: "bounceRate" },
        { name: "averageSessionDuration" },
        { name: "conversions" },
      ],
      dimensionFilter: {
        filter: {
          fieldName: "sessionDefaultChannelGroup",
          stringFilter: { matchType: "EXACT", value: "Organic Search" },
        },
      },
    },
  });
  // ... parse and return rows
}

The heart of the dashboard. Calculates current-vs-prior period trends, top queries/pages, daily sparklines, and the weighted health score.

src/app/api/admin/seo/overview/route.ts TypeScript 
// Key calculations:

// 1. Compare current (7d) vs prior (7-14d) periods
const current = await prisma.gscPageMetric.aggregate({
  where: { date: { gte: sevenDaysAgo } },
  _sum: { clicks: true, impressions: true },
  _avg: { ctr: true, position: true },
});

// 2. Trend = ((current - prior) / prior) * 100
const calcTrend = (curr, prev) =>
  prev === 0 ? 0 : ((curr - prev) / prev) * 100;

// 3. Health score (weighted)
const healthScore = Math.round(
  auditScore * 0.3 +  // avg page audit score
  ctrScore   * 0.2 +  // CTR / 5% * 100
  posScore   * 0.2 +  // 100 - (pos - 1) * 5
  errorScore * 0.15 + // 100 - unresolved404s * 2

16 Cron Sync

Daily cron job at 4 AM UTC. Protected by CRON_SECRET Bearer token. Runs GSC and GA4 syncs independently, logging status to SeoSyncLog.

src/app/api/cron/seo-sync/route.ts TypeScript 
// Vercel cron config in vercel.json:
// { "crons": [{ "path": "/api/cron/seo-sync", "schedule": "0 4 * * *" }] }

export async function GET(request: Request) {
  // 1. Verify CRON_SECRET Bearer token
  // 2. Check Google is configured

  // GSC Sync (last 3 days)
  const queries = await fetchSearchAnalytics(gscSiteUrl, dateRange);
  for (const row of queries) {
    await prisma.gscSearchQuery.upsert({
      where: { date_query_page_country_device: { ... } },
      update: { clicks, impressions, ctr, position },
      create: { ... },
    });
  }

  // GA4 Sync (yesterday)
  const rows = await fetchOrganicTraffic(ga4PropertyId, dateRange);
  for (const row of rows) {
    await prisma.gaOrganicMetric.upsert({
      where: { date_landingPage: { ... } },
      ...
    });
  }

  // Log sync status: RUNNING → COMPLETED or FAILED
}

17 404 Tracking Endpoint

Client-side POST endpoint with in-memory rate limiting (1 hit per IP per URL per minute). Detects bots, creates raw hit records, and upserts aggregated counts.

src/app/api/track-404/route.ts TypeScript 
// Rate limiting: Map<`${ip}:${url}`, timestamp>
// Window: 60 seconds, auto-cleanup at 10K entries

export async function POST(request: Request) {
  const { url, userAgent, referer } = await request.json();
  const normalizedUrl = url.split("?")[0].split("#")[0].toLowerCase();

  if (isRateLimited(`${ip}:${normalizedUrl}`)) return ok;

  const { isBot, botName } = detectBot(userAgent);

  // Raw hit log
  await prisma.notFoundHit.create({ data: { url, userAgent, referer, isBot, botName } });

  // Aggregated URL record
  await prisma.notFoundUrl.upsert({
    where: { url: normalizedUrl },
    create: { url, hitCount: 1, botHitCount: isBot ? 1 : 0, ... },
    update: { hitCount: { increment: 1 }, ... },
  });
}

export function JsonLd({ data }: { data: Record<string, any> }) {
  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(data) }}
    />
  );
}

export function MultiJsonLd({ schemas }: { schemas: Record<string, any>[] }) {
  return (
    <>
      {schemas.map((schema, i) => (
        <script key={i}
          type="application/ld+json"
          dangerouslySetInnerHTML={{ __html: JSON.stringify(schema) }}
        />
      ))}
    </>
  );
}

23 404 Monitoring Flow

1
Client Detection

User/bot hits a 404 page → JS sends POST to /api/track-404

2
Rate Limiting

In-memory check: 1 hit per IP per URL per minute. Rejects duplicates silently.

3
Bot Detection

User agent classified as good bot, bad bot, or human via pattern matching.

4
Database Storage

Creates NotFoundHit (raw log) + upserts NotFoundUrl (aggregate with bot/human counts).

5
Admin Review

Admin views 404s in /admin/404-monitor/ with filters by status, date, bot/human.

6
Resolution

Mark as IGNORED or create Redirect (REDIRECTED). Links NotFoundUrl.redirectId to Redirect.

24 Sync Strategy

SourceScheduleWindowUpsert Key
GSC QueriesDaily 4 AM UTCLast 3 daysdate + query + page + country + device
GSC PagesDaily 4 AM UTCLast 3 daysdate + page
GA4 OrganicDaily 4 AM UTCYesterdaydate + landingPage
Why 3 days for GSC? Google Search Console data is typically delayed 2-3 days. By syncing the last 3 days, you catch updated data for dates that were incomplete in prior syncs. The upsert pattern ensures no duplicates.
