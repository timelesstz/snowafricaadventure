### PATCH 1 — Remove `placeholderTrips` and query real DayTrip data

**File:** `src/app/(site)/tanzania-day-tours/page.tsx:30`
**Category:** Mock data (CRITICAL)
**Why:** `placeholderTrips` array ships hardcoded fake day trips to real users.

**Before:**
```tsx
const placeholderTrips = [
  { id: 1, name: "...", ... },
  // ...hardcoded trips
];
```

**After:**
```tsx
// This is a Server Component — fetch directly from Prisma
const dayTrips = await prisma.dayTrip.findMany({
  where: { published: true },
  orderBy: { createdAt: "desc" },
});

// In the JSX, replace placeholderTrips with dayTrips
// Add empty state:
if (dayTrips.length === 0) {
  return <EmptyState message="Day trips coming soon" />;
}
```

**Remember to:**
- Import `prisma` from `@/lib/prisma`
- Verify the DayTrip model has a `published` field (check schema)
- Remove the entire `placeholderTrips` const declaration
- Update the JSX to map over `dayTrips` instead of `placeholderTrips`

### PATCH 3 — Add 11 missing env vars to `.env.example`

**File:** `.env.example`
**Category:** Config (HIGH)
**Why:** New developers won't know these vars are needed. Runtime will fail silently or crash.

**Add these lines to `.env.example`:**

```env
# Database (Prisma Accelerate — required)
DATABASE_URL_ACCELERATE=prisma://accelerate.prisma-data.net/?api_key=your_key

# Google APIs
GA4_PROPERTY_ID=your_ga4_property_id
GOOGLE_SERVICE_ACCOUNT_KEY={"type":"service_account",...}
GOOGLE_SITE_VERIFICATION=your_verification_code
GSC_SITE_URL=https://your-site.com

# Site
NEXT_PUBLIC_BASE_URL=http://localhost:3000

# Web Push (VAPID)
NEXT_PUBLIC_VAPID_PUBLIC_KEY=your_vapid_public_key
VAPID_PRIVATE_KEY=your_vapid_private_key
VAPID_PUBLIC_KEY=your_vapid_public_key
VAPID_SUBJECT=mailto:your@email.com

# Analytics (optional)
ANALYTICS_ENDPOINT=https://your-analytics-endpoint.com
```

**Also remove these dead vars from `.env.example`:**
```
NEXT_PUBLIC_SITE_NAME  (not referenced anywhere)
SENTRY_DSN             (Sentry not integrated)
```

**DO NOT remove** `NEXTAUTH_SECRET`, `R2_ACCESS_KEY_ID`, `R2_BUCKET_NAME`, `R2_ENDPOINT`, `R2_PUBLIC_URL`, `R2_SECRET_ACCESS_KEY` — these are consumed internally by NextAuth and the S3 client respectively (detector false positives).

### PATCH 4 — Fix fallback image in RouteItinerary

**File:** `src/components/routes/RouteItinerary.tsx:970`
**Category:** Fallback image pattern (HIGH)
**Why:** `src || "https://r2.dev/..."` hides the fact that an image is missing by showing a specific hardcoded image.

**Before:**
```tsx
src={day.image || "https://pub-cf9450d27ca744f1825d1e08b392f592.r2.dev/wp-content/uploads/2024/07/kilitrekkers.webp"}
```

**After:**
```tsx
src={day.image || "/images/placeholder-itinerary.webp"}
```

**Or better — conditional render:**
```tsx
{day.image && (
  <Image src={day.image} alt={day.title} ... />
)}
```

### PATCH 7 — Add error handling to service worker fetch

**File:** `public/admin-sw.js:64`
**Category:** Error handling (HIGH)

**Before:**
```js
fetch(request).then((response) => {
  // ...
});
```

**After:**
```js
fetch(request)
  .then((response) => {
    // ...
  })
  .catch((error) => {
    console.error('Service worker fetch failed:', error);
    return caches.match(request);
  });
```
